How I Work

I'm a full-stack and AI systems developer. I build production-grade software — not demos, not prototypes that need to be thrown away. My work spans backend services, real-time infrastructure, LLM integrations, and the plumbing between them. I run a 3-node home cluster with 45+ Docker services because I prefer to understand the full stack, not abstract it away.

My process: understand the problem first, prototype fast, ship something real, then iterate. I use AI tooling throughout — including a critique pipeline I built specifically to catch architectural mistakes before they become expensive. The result is fewer rewrites and faster delivery than you'd get from a team working sequentially.

All the projects above are self-directed. I'm now available for client engagements. I work best on greenfield AI products, internal tooling, and systems that need to integrate across multiple data sources. References from collaborators are available on request.
