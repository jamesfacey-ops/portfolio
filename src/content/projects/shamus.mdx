## The Problem

Teachers in a typical classroom of 30 students spend 4+ hours every night grading and preparing differentiated assignments. IEP/504 compliance requires documenting that every student received an appropriate version of every assignment — a manual, error-prone process.

The tools teachers use (Kickboard for behavior, Schoology/Canvas for assignments, PowerSchool for grades) don't talk to each other. Everything lives in spreadsheets.

## What I Built

Shamus replaces all three with a single platform:

**AI Grading Engine** — Claude API grades student work overnight, including handwritten papers scanned and processed through Tika + OCR. The AI returns a score, rubric-aligned feedback, and flags for teacher review. Teachers see a queue of edge cases in the morning, not a stack of papers.

**Auto-Differentiation** — Every assignment is automatically generated in 3 tiers (grade-level, modified, alternative) with IEP/504 documentation generated alongside. Teachers approve with one click.

**Real-Time Dashboard** — Elixir/Phoenix LiveView dashboard shows live classroom status, student engagement, and today's science bucks tally. Updates in real-time without polling.

**Parent Communication** — Automated weekly summaries, incident reports, and grade notifications via Phoenix email service. Parents see what their student is working on without requiring teacher involvement.

**Task & Lesson Management** — Node.js task service with calendar integration. Sub plans auto-generate from current lesson context.

## Architecture

22 microservices behind nginx, deployed as Docker containers on a Proxmox LXC with 8GB RAM. PostgreSQL handles multi-tenant data with per-school row isolation. Redis for session management and caching. Qdrant + BGE-M3 for semantic search across resources and standards.

The system runs autonomously overnight — grading, generating differentiations, sending parent summaries — and surfaces a morning briefing for each teacher by 6am.
