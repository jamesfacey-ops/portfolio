## The Problem

Claude Code is excellent for autonomous work — but running it reliably for extended periods is harder than it looks. Rate limits hit without warning. Jobs freeze on tool calls. There's no way to queue multiple tasks or review results without watching a terminal.

I needed a system that could take a backlog of tasks and execute them one at a time, all day, without requiring supervision.

## What I Built

**Job Queue** — REST API accepts jobs (critique, build, research, general, chat). Jobs queue and execute one at a time to avoid parallel Claude API rate limits. Priority queue ensures urgent work runs first.

**Claude Code Spawner** — Each job launches a `claude` process with `--output-format stream-json`. The spawner reads the stream, updates job status in real-time, detects frozen output (no tokens for 90+ seconds), handles crashes, and retries up to 2 times before marking failed.

**Rate Limit Detection** — Parses Claude's error output for rate limit signals, backs off exponentially (1min → 2min → 4min), and resumes the queue automatically. No manual intervention needed.

**Heartbeat Protocol** — Running jobs call a heartbeat endpoint every few turns. The scheduler monitors heartbeats and can inject preempt requests or BTW messages while a job is running. Jobs checkpoint their work on preempt.

**WebSocket Dashboard** — Real-time view of queue depth, active job, job history, and cost per job. Served at orchestrator.jnkfacey.com via Cloudflare tunnel.

**Self-Hosting** — The orchestrator submits its own jobs. Planning a new feature? Submit a planning job. Need a critique? Submit a critique job. The system manages itself.

## Numbers

Running since January 2026. Processes planning cycles, critique rounds, build tasks, and research queries. Total cost tracked per job. Average job: $0.15–0.80 depending on complexity.
