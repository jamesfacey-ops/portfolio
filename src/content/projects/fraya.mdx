## What Fraya Does

Fraya is the single interface for managing the home. Instead of opening 6 different apps, you talk to Fraya.

**Calendar** — "Schedule a dentist appointment Thursday at 2pm" → creates the calendar event, checks for conflicts, sends a confirmation. Google Calendar is the source of truth; Fraya just makes it conversational.

**Budget** — Tracks spending, categorizes transactions, answers "how much did I spend on groceries this month", and alerts on unusual charges. PostgreSQL stores all transaction data; no external service has access to the actual numbers.

**Smart Home** — Controls Geeni, Sensi (thermostat), Ring, and other devices via dedicated MCP servers. "Turn off the living room lights and lock the front door" executes across multiple APIs in one request.

**Media** — "Download the new season of X" → triggers the *arr stack automation. "What's new on Plex?" → queries the Plex API and returns a formatted list.

**Email** → Triage and draft assistance. Fraya doesn't send email autonomously — it drafts for review.

## The Architecture

50+ n8n workflows handle the integrations. Each workflow is a discrete capability — budget import, calendar sync, smart home toggle, media request. The Python dispatcher identifies intent via Claude API and routes to the appropriate n8n webhook with structured parameters.

The n8n instance on fraya-lxc connects to PostgreSQL on utils-lxc for persistent storage. All smart home traffic stays on the local network. Only Cloudflare Tunnel traffic is exposed externally.

MCP servers (custom-built) provide tool interfaces for Ring, Geeni, and Sensi devices — these run as Docker containers on fraya-lxc and register as tools available to Claude during Fraya sessions.
